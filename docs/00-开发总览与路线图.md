## 开发阶段（共 7 阶段，单人预估 8-12 周）

### Phase 0: 项目骨架搭建（第 1 周）
- 目标：项目可运行，前后端通信打通
- 产出：
  - 目录结构创建完毕
  - requirements.txt 安装通过
  - FastAPI 启动，静态文件可访问
  - PyWebView 窗口可打开显示前端页面
  - .env 加载正常
- 里程碑：在浏览器/PyWebView 中看到前端首页

### Phase 1: 文件处理引擎（第 2-3 周）
- 目标：能读、能写、能 Diff
- 产出：
  - xlsx/xls/csv 读取统一为 DataFrame
  - 编码自动检测
  - Schema 提取
  - 格式降级写入（含 xls 行数截断、CSV BOM）
  - DataFrame Diff 计算
- 里程碑：上传两个不同格式文件，可在后端看到 Schema JSON

### Phase 2: Agent 核心 + 沙箱（第 3-5 周）
- 目标：自然语言 → Pandas 代码 → 安全执行 → 拿到结果
- 产出：
  - DeepSeek API 客户端封装
  - Prompt 模板（系统提示词 + Schema 分析 + 代码生成 + 重试）
  - 沙箱执行器（AST 扫描 + 受限命名空间 + 超时）
  - 重试回路（报错回喂 LLM，最多 3 次）
  - 脏数据检测与挂起状态
- 里程碑：上传文件 + 输入"删除金额小于0的行"，后端自动生成代码并执行成功

### Phase 3: 前端界面（第 5-6 周）
- 目标：完整的三栏交互界面
- 产出：
  - 三栏布局 + 深色/浅色主题
  - 文件拖拽上传组件
  - 对话输入区 + 消息列表
  - Diff 对比视图（红绿黄高亮）
  - 实时状态流（WebSocket 推送 Agent 思维链）
  - 异常确认弹窗
- 里程碑：全流程可用——上传→对话→Diff→确认导出

### Phase 4: 记忆系统（第 7-8 周）
- 目标：技能沉淀与复用
- 产出：
  - SQLite 表结构 + CRUD
  - ChromaDB 向量存储
  - 技能保存流程（用户确认后自动打包）
  - 技能匹配流程（上传新文件时检索相似 Schema）
  - 一键调用历史技能
  - 转换历史列表
- 里程碑：完成一次转换后保存为技能，下次上传同类文件自动匹配

### Phase 5: 集成测试 + 验收（第 9-10 周）
- 目标：通过验收标准
- 产出：
  - 两份格式完全不同的测试数据
  - 单元测试覆盖核心模块
  - 端到端验收测试
  - Bug 修复
- 里程碑：用两份异构企业系统导出表，通过自然语言指令成功转换为第三种格式

### Phase 6: 打包与交付（第 11-12 周）
- 目标：可分发执行文件
- 产出：
  - PyInstaller 打包配置
  - 安装/使用说明
  - Prompt 策略文档
  - 沙箱安全说明
- 里程碑：在未安装 Python 的 Windows 机器上启动运行

## 开发约定

1. **每个 Phase 完成后做一次手动回归测试**，不要等到 Phase 5 再测
2. **所有 LLM 调用必须记录日志**（输入 Prompt、输出结果、耗时、Token 消耗）
3. **沙箱执行的代码必须留存副本**（用于后续技能沉淀和审计）
4. **不提前优化**——先用最简单方案跑通，再重构
5. **Git 提交粒度**：每个可工作的功能点提交一次
