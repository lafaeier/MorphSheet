# 03 — 前端设计与组件

## 技术方案

- **Vue 3 CDN**：`<script src="https://unpkg.com/vue@3/dist/vue.global.prod.js"></script>`
- **无构建工具**：所有 JS 为 ES Module 或 IIFE，直接通过 `<script>` 标签加载
- **CSS 变量**：实现深色/浅色双主题，用户点击切换按钮切换 `[data-theme]` 属性
- **通信**：REST 走 `fetch()`，实时状态走 `WebSocket`

## 各组件实现要点

### 1. uploader.js — 文件上传组件

功能：
- 拖拽区域（drag & drop），检测 `.xlsx/.xls/.csv`
- 拖入时高亮边框，离开时恢复
- 选择文件后立即调用 `POST /api/upload`
- 上传中显示进度动画
- 上传完成展示 Schema 摘要（列名列表、行数）
- 随后触发技能匹配：`POST /api/match-skills`
- 如果匹配到技能，在界面顶部弹出提示"检测到相似历史转换模板，是否一键应用？"

关键实现细节：
- 使用 `FormData` 包装文件，`fetch()` 上传
- 拖拽事件：`dragover`（阻止默认行为+高亮）、`dragleave`（取消高亮）、`drop`（取文件）
- 支持点击打开文件选择器作为 fallback
- 文件大小前端校验（>50MB 提示）

### 2. chat.js — 对话区组件

功能：
- 消息列表（用户消息靠右蓝色气泡，系统消息靠左灰色气泡）
- 输入框 + 发送按钮
- 支持预设指令快捷按钮（"删除空行"、"去除重复行"、"金额列取绝对值"）
- 发送自然语言指令→调用 `POST /api/convert`
- 根据返回状态：
  - `awaiting_confirmation` → 切换到 Diff 视图
  - `awaiting_human_confirmation` → 弹出确认弹窗
- 消息自动滚动到底部

关键实现细节：
- 每条消息格式：`{ role: 'user'|'system', content: string, timestamp, extra?: { phase, issues } }`
- 发送请求时输入框禁用，显示 loading 状态
- 错误消息显示为红色系统消息

### 3. diff-view.js — Diff 对比视图（加分项核心）

功能：
- 左右分屏：左屏"源数据"，右屏"转换后"
- 同步滚动（左屏滚动时右屏跟随，反之亦然）
- 数据修改处高亮：
  - 绿色背景：新增的列
  - 红色背景：被删除的行
  - 黄色背景：被修改的单元格
- 顶部显示变更摘要（"新增 3 列，删除 12 行，修改 45 个单元格"）
- "确认导出"按钮和"放弃"按钮

关键实现细节：
- 数据从 `POST /api/convert` 返回的 `diff` 字段获取
- 使用两个 `<table>` 并排，CSS `overflow-y: auto; max-height` 实现可滚动
- 同步滚动：监听 `scroll` 事件，设置对方 `scrollTop`
- 高亮实现：给 `<tr>` 或 `<td>` 添加 `.added` / `.removed` / `.modified` 类
- 渲染优化：只渲染可视区域（简单场景不需要虚拟滚动，500行以内直接渲染）

### 4. sidebar.js — 左侧边栏

功能：
- 两个 Tab：历史记录 / 技能库
- 历史记录：
  - 按时间倒序，每条显示：源文件名、目标格式、指令摘要、时间、状态图标
  - 点击可查看详情（展示当时的 Diff 快照，只读）
- 技能库：
  - 卡片式布局，每张卡片显示：技能名、描述、使用次数、创建时间
  - 拖拽技能卡片到主工作区 → 自动填入技能（或点击"应用"按钮）
  - 右键菜单：删除技能
  - 匹配到的技能高亮边框+星标

关键实现细节：
- 数据通过 `GET /api/history` 和 `GET /api/skills` 获取，在 sidebar 组件 `setup()` 中请求
- 技能应用：将 `skill_id` 传入 `POST /api/convert` 的 `use_skill_id` 字段

### 5. status-panel.js — 右侧状态面板

功能：
- 实时显示 Agent 思维链步骤
- 当前步骤高亮，已完成步骤打勾
- 显示步骤时序（每个步骤的时间）
- 异常状态：面板变红色底色，弹窗高亮问题行和修改建议
- 显示 Token 消耗统计（如果后端提供）

关键实现细节：
- 通过 WebSocket 接收 `phase` 类型消息更新步骤
- 步骤列表：`analyzing_schema → mapping_fields → generating_code → executing → diff_ready → exported`
- 每个步骤有 3 种状态：`pending` | `active` | `done` | `error`

### 6. confirm-modal.js — 异常确认弹窗

功能：
- 模态弹窗，背景半透明遮罩，点击遮罩不关闭
- 显示异常行详情：
  - 行号、列名、当前值（红色高亮）
  - 错误描述
  - 建议修改操作
  - 建议修改后的值预览
- 操作按钮：
  - "采纳建议" — 调用 `POST /api/confirm-action` with `action: "accept_suggestion"`
  - "跳过此行" — `action: "skip_row"`
  - "手动修改" → 展开编辑框，用户修改值后提交（`action: "accept_suggestion"` + `overrides`）
  - "终止转换" — `action: "abort"`

### 7. toast.js — 通知组件

功能：右上角弹出短暂通知，3秒后自动消失
类型：成功（绿）、错误（红）、警告（黄）、信息（蓝）

## API 通信层（api.js）

```javascript
// 封装所有 REST 请求
const API = {
  async upload(file) {
    const fd = new FormData();
    fd.append('file', file);
    const res = await fetch('/api/upload', { method: 'POST', body: fd });
    if (!res.ok) throw await res.json();
    return res.json();
  },

  async setTarget(targetSpec) {
    const res = await fetch('/api/set-target', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(targetSpec)
    });
    if (!res.ok) throw await res.json();
    return res.json();
  },

  async convert(fileId, instructions, skillId = null) {
    const res = await fetch('/api/convert', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ file_id: fileId, instructions, use_skill_id: skillId })
    });
    if (!res.ok) throw await res.json();
    return res.json();
  },

  async confirmAction(taskId, action, overrides = null) {
    const res = await fetch('/api/confirm-action', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ task_id: taskId, action, overrides })
    });
    if (!res.ok) throw await res.json();
    return res.json();
  },

  async export_(taskId, saveAsSkill = false, skillName = null) {
    const res = await fetch('/api/export', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ task_id: taskId, save_as_skill: saveAsSkill, skill_name: skillName })
    });
    if (!res.ok) throw await res.json();
    return res.json();
  },

  async getHistory(limit = 20, offset = 0) {
    const res = await fetch(`/api/history?limit=${limit}&offset=${offset}`);
    return res.json();
  },

  async getSkills(limit = 20) {
    const res = await fetch(`/api/skills?limit=${limit}`);
    return res.json();
  },

  async matchSkills(fileId) {
    const res = await fetch('/api/match-skills', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ file_id: fileId })
    });
    return res.json();
  },

  getDownloadUrl(fileId) { return `/api/download/${fileId}`; }
};
```

## WebSocket 连接管理（ws.js）

```javascript
class AgentWS {
  constructor() {
    this.ws = null;
    this.listeners = [];
  }

  connect(taskId) {
    const protocol = location.protocol === 'https:' ? 'wss:' : 'ws:';
    this.ws = new WebSocket(`${protocol}//${location.host}/ws/agent-status/${taskId}`);
    this.ws.onmessage = (e) => {
      const data = JSON.parse(e.data);
      this.listeners.forEach(fn => fn(data));
    };
    this.ws.onclose = () => { /* 自动重连逻辑（可选） */ };
    this.ws.onerror = () => { /* 错误处理 */ };
  }

  onMessage(fn) { this.listeners.push(fn); }

  disconnect() {
    if (this.ws) { this.ws.close(); this.ws = null; }
  }
}
```
