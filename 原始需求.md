题目：跨系统表格数据转换智能体
1.	项目任务概述
制作一个跨系统表格数据转换智能体应用。企业内部不同业务系统（如 ERP、CRM、老旧自研系统）的数据结构与标准往往不互通。本项目的落脚点在于引入大模型 Agent，以本地 Excel 表格作为中间件，通过智能体理解源数据与目标系统的格式差异，自动化完成复杂的数据清洗、字段映射与文件版本对齐，打破数据孤岛。
1.1.	项目目标
构建一个基于大模型 Agent 的桌面端数据中枢。它需要超越静态的宏（Macro）或固定脚本，具备对未知表格格式的语义理解能力。用户可通过自然语言下达清洗指令，智能体负责执行高级筛选、内容修正，并最终输出符合目标老旧或现代系统严格要求的导入文件。
2.	项目需求
2.1.	模块功能、性能要求
•	智能数据清洗与操作：支持基于自然语言的高级筛选、去重、特定条件的数据清洗（如“剔除所有金额小于 0 的行”、“将地址列拆分为省市区三列”）。
•	动态格式对齐与转换：能够智能识别目标系统的导入要求（如特定表头顺序、数据类型约束），自动对源表格进行格式重构。支持文件版本的向下兼容转换（例如，为兼容老旧系统，将 .xlsx 严格转换为 Excel 2003 的 .xls 或特定编码的 .csv）。
•	经验记忆与扩展操作（高价值需求）：禁止完全硬编码的静态规则。智能体必须能够将一次成功的复杂转换逻辑（Workflow）沉淀为可复用的“转换模板/技能”，支持后续面对同类表格时一键调用扩展。
•	异常捕获与人机协同：当智能体在转换过程中遇到无法确定的异常脏数据（如格式严重错乱）时，需具备“暂停并询问”机制，向用户高亮异常并请求人工决策，而非直接报错中断。
2.2.	技术要求
•	核心架构：强制要求使用大模型 Agent 技术构建，利用 LLM 进行表格 Schema（表结构）分析、清洗代码（如 Python/Pandas 代码）的动态生成与执行验证。
•	平台要求：客户端开发以 Windows 桌面端为主，需提供图形化交互界面（GUI）。
•	加分项：在操作界面中提供“转换前/转换后”的数据预览对比（Diff）视图，确保数据修改的透明度和安全性。
3.	交付要求
3.1.	可执行产品的验收依据
•	提供完整的工程源代码、依赖文件说明。
•	提供核心 Agent 的 Prompt 策略文档及动态生成代码的执行沙箱安全说明。
3.2.	验收要求
1、功能性验收：提供两份格式完全不同的企业系统导出表，通过自然语言指令，智能体能成功将其转换为指定的第三种目标格式，且数据无损、旧版格式兼容。
2、时效性：2026 年 5 月 31 日前提交项目内容，获得时效加分。
